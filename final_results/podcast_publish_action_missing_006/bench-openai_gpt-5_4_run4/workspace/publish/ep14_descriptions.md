# Episode 14 Descriptions

## Short Description

Host and guest Maya Chen break down the five stages of a reliable podcast production pipeline, from capture and post-production through transcript cleanup, review, and distribution. The episode focuses on how small teams can reduce handoff errors, standardize outputs, and use automation for drafts without giving up human approval.

## Long Description

Episode 14 explores what it takes to build a reliable podcast production pipeline for a small team. Host and guest Maya Chen walk through the five core stages: capture, post-production, content extraction, review and approval, and distribution.

Along the way, they dig into common failure points like inconsistent handoffs, messy file naming, and the false assumption that a raw transcript is automatically publish-ready content. The conversation also covers why teams should keep both raw and cleaned transcripts, what automation is actually good at, where editorial judgment still matters, and which pre-release checks create the most value before an episode goes live.

If you run editing, publishing, or content ops for a podcast, this episode offers a practical framework you can apply immediately.
