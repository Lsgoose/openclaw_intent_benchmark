# Episode 14: Building a Reliable Podcast Production Pipeline with Maya Chen

## Timestamps

**[00:00:00] Introduction to podcast production pipelines**
- Host introduces the topic: how a show goes from raw recording to polished episode with clean audio, transcript, chapter markers, social clips, title, description, and metadata
- Guest Maya Chen joins to discuss production systems for media teams and creator businesses

**[00:00:47] The pipeline as the product**
- Maya explains that people often think the product is just the conversation, but the way it gets edited, packaged, labeled, and distributed matters a lot
- If title is vague, transcript is sloppy, chapter markers are off, or publishing checklist is inconsistent, small mistakes stack up

**[00:01:20] Five stages of podcast production pipeline**
1. **Capture**: Recording conversation and collecting companion materials (guest bios, sponsor copy, links, references)
2. **Post-production**: Cleaning audio, trimming mistakes, deciding what stays in
3. **Content extraction**: Transcript generation, summaries, quotes, show notes, chapter markers, titles, clips
4. **Review and approval**: Checking facts, sponsor language, names, links, editorial/legal issues
5. **Distribution**: Final package goes to hosting platform, newsletter, social channels, archives

**[00:02:13] Pipeline as software build analogy**
- Source material, transformations, validation steps, packaging, and deployment
- Pipelines become fragile when process depends too much on memory and heroics

**[00:02:33] Common pain points for small teams**
1. **Handoffs**: One person records, another edits, someone else writes description - no single source of truth
2. **Inconsistent naming and asset storage**: Files like "final_v2_real_final.wav" create confusion
3. **Transcript-driven tasks**: Raw transcript is not automatically publish-ready copy

**[00:03:09] Transcripts as rich input, not final truth**
- Transcripts contain filler, false starts, jokes that don't read well, corrected statements
- Treating transcript as clean structured truth leads to low-quality or misleading derivative content
- Transcripts are excellent for first drafts of summaries, chapter suggestions, quote candidates, social post options
- Anything authoritative (facts, links, sponsor language, names, affiliations, dates, claims) needs human review

**[00:04:26] Lightweight pipeline for small teams**
- One folder or workspace per episode
- Clear naming convention
- Checklist with defined handoff points
- Fixed output bundle: final audio, clean transcript, show notes, title, description, chapter list, thumbnail, publish log

**[00:04:54] Raw vs. clean transcripts**
- Keep raw transcript for traceability and internal reference
- Cleaned transcript removes disfluencies, fixes speaker labels, formats for readability
- Cleaned version becomes preferred source for summarization and repurposing

**[00:05:49] Automation pitfalls**
- Overestimating editorial judgment: agents can propose titles but don't know audience positioning
- Overestimating exception handling: edge cases (guest name corrections, legal concerns, sensitive anecdotes, sponsor constraints) break fully automated systems
- Healthy mindset: Use automation for acceleration, not abdication

**[00:06:52] Title strategy**
- Titles do a lot of work - they're a contract with the listener
- Generate multiple title candidates from different angles: problem-oriented, guest-oriented, outcome-oriented, curiosity-driven
- Compare against actual audience promise
- Frequency is not strategy: most repeated phrase often isn't most compelling framing

**[00:08:13] High-value pre-publish checks**
1. Names and affiliations (guest name, title, company)
2. Links and references (verify destinations)
3. Sponsor compliance (required language present)
4. Sensitive edits (confirm requested removals)
5. Metadata consistency (title, description, transcript, chapter list, social copy all describe same episode)

**[00:09:13] Practical takeaways for improving pipeline**
1. Define standard episode folder structure and naming convention
2. Create publish checklist with single owner
3. Separate raw transcript and cleaned transcript
4. Define required output bundle for every episode
5. Keep lightweight episode log to track delays, confusion, repeated errors

**[00:09:58] Automation principle**
- Make the machine produce options
- Make the human approve what becomes public
