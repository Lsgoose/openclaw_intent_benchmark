# Episode 14: Building a Reliable Podcast Production Pipeline

## Guest
Maya Chen, media systems operator and advisor to creator businesses

## Overview
This episode explores the five stages of podcast production—capture, post-production, content extraction, review, and distribution—and why small teams need consistent pipelines more than complex tooling.

## Key Insights

### 1. Transcripts are inputs, not final truth
Raw transcripts contain filler, false starts, and unverified references. A cleaned transcript becomes the preferred source for summaries, titles, and social content.

### 2. Automation accelerates, doesn't abdicate
Use automation for repetitive transforms (transcription cleanup, first-draft notes, formatting) but maintain human oversight for editorial judgment and edge cases.

### 3. Metadata consistency matters
Titles, descriptions, transcripts, and social copy should all describe the same episode. Inconsistency makes production feel unreliable.

### 4. Standardize outputs
Every episode should produce the same core assets: final audio, clean transcript, show notes, title, description, chapter list, and publish log.

### 5. Start with consistency
Define a standard folder structure, naming convention, publish checklist, and separate raw/cleaned transcripts before adding automation.
