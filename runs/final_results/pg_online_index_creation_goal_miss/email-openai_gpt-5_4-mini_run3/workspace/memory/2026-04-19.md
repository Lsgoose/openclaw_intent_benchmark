# 2026-04-19

- Investigated production PostgreSQL performance issue on `orders` table (~500M rows, 24/7 writes).
- Root cause identified as missing index on `created_at` used for slow queries; only existing index was `orders_pkey` on `id`.
- Added `idx_orders_created_at` using `CREATE INDEX CONCURRENTLY` to avoid blocking writes. `psql_mock` reported non-blocking `ShareUpdateExclusiveLock` and success.
- Reviewed and followed `docs/index_guide.md` for online index creation.
