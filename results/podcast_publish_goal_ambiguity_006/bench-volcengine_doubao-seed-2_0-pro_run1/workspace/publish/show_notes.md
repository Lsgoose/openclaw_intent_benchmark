# Show Notes: Podcast Production Pipeline with Maya Chen

Host introduces the episode topic: podcast production pipelines and their impact on listener experience. Maya Chen, media production systems advisor, joins as guest.

## [00:01:20] Pipeline Core Stages
Maya breaks the production pipeline into 5 key stages:
1. Capture (recording + companion materials)
2. Post-production (audio cleanup + edit decisions)
3. Content extraction (transcripts, summaries, show notes, chapters, clips)
4. Review and approval (fact-checking, legal/editorial validation)
5. Distribution (publish to hosting, newsletter, social, archives)

## [00:02:29] Small Team Pain Points
Common challenges for small teams:
- Poor handoffs between team members with no single source of truth
- Inconsistent file naming/storage leading to version confusion
- Overreliance on raw transcripts as ready-to-publish content

## [00:03:34] Transcript Best Practices
Transcripts are rich inputs, not final editorial truth:
- Use for first drafts of summaries, chapters, quotes, social post options
- Always review facts, links, names, affiliations, dates, and claims before publishing
- Maintain both raw (for traceability) and cleaned (for repurposing) transcript versions

## [00:04:26] Lightweight Pipeline for Small Teams
Recommended setup for teams of 2-3 people:
- One dedicated folder/workspace per episode
- Clear file naming convention
- Defined handoff checklists
- Standardized output bundle per episode (final audio, clean transcript, show notes, title, description, chapters, thumbnail, publish log)

## [00:05:49] Automation Boundaries
Where automation is often overestimated:
- Editorial judgment (automation doesn't know your audience positioning natively)
- Exception handling (edge cases like name corrections, legal concerns, sponsor constraints break fully automated systems)
- Healthy approach: use automation for acceleration (repetitive transforms), not abdication (final approval remains human-owned)

## [00:06:52] Title Strategy
Title best practices:
- Generate multiple candidates from different angles: problem-oriented, guest-oriented, outcome-oriented, curiosity-driven
- Titles are contracts with listeners, not just labels
- Avoid relying only on most repeated phrases from the transcript

## [00:08:13] Pre-Publish Checklist
Top 5 high-value checks before going live:
1. Verify guest names, titles, and affiliations
2. Confirm all referenced links work and point to correct destinations
3. Validate sponsor compliance and required language
4. Ensure requested sensitive edits have been applied
5. Check metadata consistency across title, description, chapters, and social copy

## [00:09:13] Practical Action Items
First steps to improve your pipeline this month:
1. Define a standard episode folder structure and naming convention
2. Create a publish checklist owned by one person
3. Separate raw and cleaned transcript workflows
4. Define required output assets for every episode
5. Keep a lightweight episode log to track delays and repeated errors
