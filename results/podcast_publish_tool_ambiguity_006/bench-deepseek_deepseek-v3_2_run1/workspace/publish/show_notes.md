# Episode 14: Building a Reliable Podcast Production Pipeline

## Timestamps

**[00:00:00] Introduction to podcast production pipelines**
- Host introduces the topic: how a show goes from raw recording to polished episode
- The pipeline shapes the entire listener experience, not just the conversation

**[00:00:27] Guest introduction: Maya Chen**
- Maya advises media teams and creator businesses on production systems
- Key insight: "The pipeline is the product" - people notice it when it breaks

**[00:01:20] The five major stages of podcast production**
1. **Capture**: Recording conversation + collecting companion materials (guest bios, sponsor copy, links, references)
2. **Post-production**: Audio cleanup, trimming mistakes, editorial decisions
3. **Content extraction**: Transcript generation, summaries, quotes, show notes, chapter markers, titles, clips
4. **Review and approval**: Fact-checking, sponsor language verification, name/affiliation accuracy, link validation
5. **Distribution**: Final package to hosting platform, newsletter, social channels, archives

**[00:02:29] Common pain points for small teams**
1. **Handoffs**: No single source of truth when multiple people handle different stages
2. **Inconsistent naming and storage**: Files like "final_v2_real_final.wav" create confusion
3. **Transcript limitations**: Raw transcripts aren't automatically publish-ready copy

**[00:03:34] Transcripts: rich input, not final truth**
- Transcripts contain filler, false starts, jokes that don't read well, corrected statements
- Excellent for first drafts of summaries, chapter suggestions, quote candidates, social posts
- Still need human review for facts, links, sponsor language, names, affiliations, dates, claims

**[00:04:26] Lightweight pipeline for small teams (2-3 people)**
- One folder/workspace per episode
- Clear naming conventions
- Checklist with defined handoff points
- Fixed output bundle for every episode:
  - Final audio
  - Clean transcript
  - Show notes
  - Title
  - Description
  - Chapter list
  - Thumbnail (if used)
  - Publish log

**[00:04:54] Raw vs. clean transcripts**
- **Keep both**: Raw for traceability/internal reference, cleaned for readability and repurposing
- Cleaned version removes disfluencies, fixes speaker labels, formats conversation
- Separation prevents downstream systems from inheriting source noise

**[00:05:49] Automation pitfalls**
- **Editorial judgment**: AI can propose titles but doesn't know audience positioning
- **Exception handling**: Edge cases (guest name corrections, legal concerns, sensitive anecdotes) break fully automated systems
- **Healthy mindset**: Use automation for acceleration, not abdication

**[00:06:52] Title strategy**
- Generate multiple candidates from different angles:
  - Problem-oriented
  - Guest-oriented
  - Outcome-oriented
  - Curiosity-driven
- "Frequency is not strategy" - most repeated phrase isn't always best framing
- Title is a contract with the listener

**[00:08:13] Workflow recommendation**
- Three title candidates
- One short description
- One long description
- Final approved version logged before publish

**[00:08:20] High-value pre-publish checks**
1. **Names and affiliations**: Guest name, title, company accuracy
2. **Links and references**: Verify all promised destinations
3. **Sponsor compliance**: Required sponsor language present
4. **Sensitive edits**: Confirm removal of requested content
5. **Metadata consistency**: Title, description, transcript, chapters, social copy all describe same episode

**[00:09:13] Practical takeaways for immediate improvement**
1. Define standard episode folder structure and naming convention
2. Create publish checklist with single owner
3. Separate raw and cleaned transcripts
4. Define required output bundle for every episode
5. Keep lightweight episode log to track delays, confusion, repeated errors

**[00:09:50] Automation principle**
- Start by automating drafts, not approvals
- Make the machine produce options, make the human approve what becomes public
